# Spec 1: Mise Mockup Cleanup & Refinement

## Scope

Update the existing 50 HTML/CSS mockups in `output/screens/` based on a detailed review of every view. This means:
- **Delete** views that were rejected
- **Modify** views that were kept but need changes
- **Create** new views that were identified as missing (sign-in, offline states)
- Keep using shadcn UI kit, same shared.css, same mobile-first static HTML approach as spec-0

This is a **UI-only** spec. No backend, no Firebase wiring, no LLM integration.

---

## Architectural Decisions (from QA)

- **Offline mode**: Read-only for recipes. Shopping list works offline (it's purely local storage).
- **Shopping list storage**: Purely local, not synced to Firebase. No cross-device sync. Discarded on sign-out or account switch — this is acceptable.
- **LLM tool harness**: Create, read, update, list (no delete for recipes). Security hardening deferred to backend implementation.
- **Tool call display**: Ephemeral one-liner UX indicator only (e.g. "Searching recipes..."). Not a confirmation mechanism, not a diff view. Informational and ephemeral.
- **Recipe card view**: Read-only display. Recipes can only be created, edited, or imported via the chat.
- **Auth**: Firebase sign-in required. No anonymous sessions. On sign-out, local data (shopping list) is discarded.
- **Units**: Metric only for now.

## Global Design Rules

These apply across ALL views:

- **No avatars** — remove all avatar circles from AI and user messages
- **No tabs** — everything is single-viewport scrollable sections, no tab navigation
- **Ingredient layout**: ingredient name on the LEFT, quantity on the RIGHT (this was inverted in many views — fix everywhere)
- **Send button**: only visible when user has started typing (Gemini-style), hidden in empty state
- **Suggestion bubbles**: always positioned right above the message input bar, not inline in the chat
- **Checkboxes**: consistent rounded-square style across the entire app
- **No "In pantry" labels or badges** anywhere
- **No strikethrough** on deselected ingredients — just grey out the whole line (ingredient + quantity)
- **Floating header bar**: back arrow (top-left), title (center), shopping cart icon (top-right) — reuse across recipe card view and shopping list view
- **Cook Mode bubble**: display all steps top-to-bottom in one viewport, no next/back pagination. Inline ingredient quantities within each step. Tips (lightbulb tooltips) go inside the step, not outside.
- **Glossary tooltips**: keep them — highlighted culinary terms with a tooltip popup. But remove the "culinary terms in this recipe" summary section.

## LLM Tool Call Display

Add to relevant chat views: a discrete, ephemeral one-liner shown while the LLM is using tools. Examples:
- "Searching recipes with keyword chicken..."
- "Updating recipe Hainanese chicken..."
- "Adding items to shopping list..."

This text appears below the thinking indicator (three dots), is styled in small grey text, and disappears when the tool call completes. It does NOT persist in chat history.
